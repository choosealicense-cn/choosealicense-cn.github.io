开源许可证可保护贡献者和用户。没有这种保护，企业和精明的开发人员就不会接触项目。

{ 以下哪项最能描述您的情况？ }

  • 我需要在软件开发社区工作。

    使用您所贡献或依赖的社区所偏爱的许可证。您的项目将完全适合。

    如果您的依赖项没有许可证，请要求其维护者添加许可证。

  • 我希望协议简单而宽松。

    MIT 许可证特别简短且合适。它让人们几乎可以对您的项目进行任何操作，例如编写和分发封闭源代码版本。

    {% include using-sentence.html license-id="mit" %}

  • 我关心改进可以共享。

    GNU GPLv3 许可证 也让人们做几乎任何他们想要与您的项目，但禁止分发封闭源代码的版本。

    {% include using-sentence.html license-id="gpl-3.0" %}

{ 如果这些都不适合我怎么办？ }

  • 我的项目不是软件。

    针对这些项目的许可证。

  • 我想要更多选择。

    有更多许可证可用。

  • 我不想选择许可证。

    如果不选许可证，会有这样的后果。
